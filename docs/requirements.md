# 需求文档 - 高德地图行程规划

## 概述

基于高德地图数据，构建一个手机端优先的网页工具，帮助用户规划"吃、住、行"行程。

## 用户角色

普通用户（无需登录，用完即走）

## 功能需求

### 模块一：吃什么（餐厅推荐）

| 输入项 | 说明 |
|--------|------|
| 预算范围 | 最低 ¥\_\_ 最高 ¥\_\_（人民币） |
| 用餐时段 | 早餐(6-10点) / 午餐(11-14点) / 晚餐(17-21点) / 夜宵(21点后) |
| 地点 | 文本输入 + 高德模糊搜索，用户点击确认 |
| 菜系风格 | 可选输入，如"川菜"、"日料"、"火锅"、"粤菜" |

**功能**：根据用户条件，调用高德地图 POI 搜索，返回餐厅列表，用户可勾选 1~3 家。

### 模块二：去哪里（目的地搜索）

| 输入项 | 说明 |
|--------|------|
| 目的地 | 文本输入 + 实时模糊搜索建议 |

**功能**：用户输入目的地名称，实时弹出搜索建议列表，选中后可继续添加多个目的地。支持删除已选目的地。

### 模块三：住哪里（酒店推荐）

此模块分两小步：

**3a. 智能推荐住宿区域**
- 基于模块一和模块二已选地点的地理位置
- 计算地理中心点
- 搜索附近地铁站
- 计算各地铁站到各目的地的公交/地铁路线
- 按换乘次数少、总耗时短排序
- 推荐 3-5 个地铁站区域

**3b. 按价格筛选酒店**
- 用户选定住宿区域后，输入价格区间（最低 ¥\_\_ 最高 ¥\_\_）
- 调用高德 POI 搜索，类型=酒店
- 用户选择 1 家酒店

### 模块四：生成行程

- 汇总前三步所有选择
- 生成简洁列表格式的行程
- 包含出行提示（地铁路线、换乘信息）
- 支持复制到剪贴板
- 支持重新规划

## 非功能需求

- 手机端优先的响应式设计
- 纯前端，无需后端和数据库
- 淡蓝色主色调，简洁直观的 UI
- 每次使用数据不保存
